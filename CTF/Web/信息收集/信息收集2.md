1. 总有人把后台地址写入*robots*，帮黑阔大佬们引路
    * robots.txt 一个非常重要得到文件，通常情况下，主要用于指定搜索引擎蜘蛛spider在网站里的抓取范围,用于声明蜘蛛不可以抓取哪些网站资源及可以抓取哪些网站资源。
    * 查看robots.txt文件，直接访问url/robots.txt获得flag
1. phps源码泄露有时候能帮上忙   
   * phps文件泄露，直接访问index.phps
1. 解压源码到当前目录，测试正常，收工
    * dirsearch脚本的初步使用与解压源码 
    * dirsearch:一个用字典暴力枚举 Web 目录 / 文件的 Python 脚本;
    * dirsearch示例  `python3 dirsearch.py -u (http://challenge-91f1f5e6a791ab02.sandbox.ctfhub.com:10080/) -e *`括号内放网址
    * dirsearch扫描可以看到www.zip
    * 直接访问url/www.zip,获得flag(注意解压看到的flag不一定都是正确的,知道文件名不如直接访问一下)
    * www.zip:把整站源码打包备份成 www.zip 放在网站根目录，却没做好访问控制，结果你可以直接通过 URL 把它下回来。
    * *先扫描，后访问，*得到对应的flag信息。

1. 发现网页有个错别字？赶紧在生产环境vim改下，不好，死机了（vim缓存泄露）
   * 本题目的是让答题者了解到vim编辑器使用时会有一个缓存文件，这个缓存文件在保存时进行删除，但是在vim意外退出时这个缓存文件，会以源文件加一个.swp后缀进行保存，我们可以尝试访问这些文件来尝试获取一些文件。
   * 以 index.php 为例 
   * 第一次产生的缓存文件名为 .index.php.swp
   * 第二次意外退出后，文件名为.index.php.swo
   * 第三次产生的缓存文件则为 .index.php.swn
   * 注意：index前有 " . "
   * 直接url/index.php.swp

1. 域名其实也可以隐藏信息，比如flag.ctfshow.com 就隐藏了一条信息 
   *  直接使用命令： nslookup -qt=txt flag.ctfshow.com 
   *  其中有： 非权威应答: flag.ctfshow.com text ="flag{just_seesee}"
1. 有时候网站上的公开信息，就是管理员常用密码
   * 网站底部、About页面等常见地方可能包含管理员信息
   * /robots.txt 或 dirsearch
   * robots.txt 一个非常重要得到文件，通常情况下，主要用于指定搜索引擎蜘蛛spider在网站里的抓取范围,用于声明蜘蛛不可以抓取哪些网站资源及可以抓取哪些网站资源。
   * 查看robots.txt 其中`Disallow: /admin/`意思告诉爬虫不可以看admin
   * 用户名admin。密码：在页面的最下方number
